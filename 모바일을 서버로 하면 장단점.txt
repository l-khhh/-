앱의 개발 방식에 따른 분류
- native app 	android- java //ios- objective-c로 작성
		전문 통신으로 구성, xml or json데이터를 기준으로 각종 요청사항 처리, 서버측은 화면 처리에 대한 기준 필요없음

- hy브리드 app	html5, css3, js를 이용해 작성, native에서는 webview or uiwebview를 띄어서 로컬 리소스 주소 넣고 구동
		전문 통신으로 구성, xml or json 데이터를 기준으로 각종 요청사항 처리, 서버 측은 화면 처리에 대한 기준 필요없음

- 웹 app		서버 측 기술을 이용하여 반응형 웹으로 구성, native에서는 웹뷰 or유아이웹뷰를 띄어서 서버 측 주소를 넣고 구동
		일반 웹과 동일하게 화면 처리까지 포함, 클라이언트 사이드에서 전문처리 일부 필요, 반응형 웹 기술 필요


왜 웹 app선택?

1. 반응형 웹으로
2. 브라우저만 있다면 아이폰, 갤럭시 상관없이 구동되며
3. 실시간 수정이 최대 장점
4. 네이버와 항공사어플들이 이렇게 만들며
5. 최근에는 native와 web을 섞거나 hybrid와 web을 섞어서 사용하는 추세이다.
6. 영남대 어플도 web섞여있음
